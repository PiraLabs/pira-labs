T: Escolha de abordagem problem-first vs tool-first
F: The Complete Guide to Building Skills for Claude.pdf
TP: framework
FN: Define se a skill deve partir do problema do usuário ou do ferramental já conectado. Use antes de escolher o padrão de orquestração, porque aqui nasce metade dos erros de desenho.
PERSONA: —
ETAPAS:
- Problem-first: partir de uma necessidade concreta e deixar a skill orquestrar as chamadas certas na sequência certa.
- Tool-first: partir do MCP já conectado e ensinar os melhores workflows e práticas para usá-lo.
- Pattern selection: escolher o padrão abaixo com base na direção dominante da skill.
REGRAS: a maioria das skills tende mais para um lado; saber esse lado evita framework errado.
SC: a abordagem escolhida precisa refletir o modo como o usuário enuncia a necessidade.
OF: —
CP: forma de entrada do usuário, maturidade do usuário com a ferramenta, grau de abstração do problema.
OT: pode envolver um ou mais MCPs.
SP: —
