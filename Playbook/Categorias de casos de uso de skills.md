T: Categorias de casos de uso de skills
F: The Complete Guide to Building Skills for Claude.pdf
TP: framework
FN: Classifica o tipo de skill em três famílias recorrentes observadas no guia. Use para decidir o padrão de construção mais adequado ao tipo de problema.
PERSONA: —
ETAPAS:
- Category 1: Document & Asset Creation: usar para criar output consistente e de alta qualidade, como documentos, apresentações, apps, designs, código e artefatos similares.
- Category 2: Workflow Automation: usar para processos multi-etapas que se beneficiam de metodologia consistente, inclusive com coordenação entre múltiplos MCP servers.
- Category 3: MCP Enhancement: usar para guidance de workflow que amplia o acesso a ferramentas fornecido por um MCP server.
REGRAS: escolher a categoria pela estrutura do trabalho, não pela aparência superficial do output.
SC: usar a categoria cujas key techniques correspondem ao workflow necessário.
OF: —
CP: tipo de output desejado, necessidade de iteração, necessidade de coordenação entre MCPs.
OT: pode usar capacidades nativas, um MCP ou múltiplos MCPs.
SP:
- Category 1 key techniques: embedded style guides, template structures, quality checklists, no external tools required.
- Category 2 key techniques: step-by-step workflow, validation gates, templates, iterative refinement loops.
- Category 3 key techniques: sequential MCP calls, embedded domain expertise, added context, common MCP error handling.
