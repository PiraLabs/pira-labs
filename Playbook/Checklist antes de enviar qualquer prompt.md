F: playbook-a-v5.md
TP: framework
FN: Faz validação mínima de especificidade, restrição, formato e contexto antes do envio. Use em qualquer prompt cujo custo de erro não seja trivial.
PERSONA: —
ETAPAS:
- Especificidade: o pedido está específico o suficiente para não precisar de adivinhação?
- Restrições: as restrições (o que NÃO fazer) estão explícitas?
- Formato: o formato de entrega está claro?
- Contexto: o contexto necessário está no prompt ou nos arquivos do projeto?
- Encadeamento: se for tarefa complexa, está dividida em etapas com checkpoint?
