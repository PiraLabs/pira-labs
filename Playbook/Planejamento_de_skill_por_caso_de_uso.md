T: Planejamento de skill por caso de uso
F: The Complete Guide to Building Skills for Claude.pdf
TP: framework
FN: Define uma skill a partir de casos de uso concretos antes de qualquer escrita técnica. Use quando a vontade de montar estrutura veio antes de provar que existe um workflow real.
PERSONA: —
ETAPAS:
- Start with use cases: identificar 2-3 concrete use cases your skill should enable.
- Trigger: registrar o que o usuário diria para ativar a skill.
- Steps: mapear o fluxo multi-etapas necessário para chegar ao resultado.
- Result: definir o resultado final esperado do workflow.
- Ask yourself: responder o que o usuário quer realizar, quais workflows isso exige, quais ferramentas serão necessárias e qual conhecimento de domínio precisa ser embutido.
REGRAS: começar pelo problema real do usuário, não pela estrutura técnica; preferir workflows repetíveis e multi-etapas.
SC: o caso de uso precisa ter trigger identificável, passos claros e resultado verificável.
OF: —
CP: objetivo do usuário, frases de ativação, passos do workflow, ferramentas necessárias, conhecimento de domínio.
OT: pode usar capacidades nativas ou MCP.
